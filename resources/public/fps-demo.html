Scittle ClojureScript JIT — FPS comparison

Same particle animation, same code, both loaded from the jsDelivr CDN. Left: interpreter. Right: JIT (compiles function bodies to JavaScript at runtime).

Each version runs in its own Web Worker, rendering to an OffscreenCanvas — genuinely parallel threads, so the frame rates are independent.

Before JIT scittle 0.8.31

With JIT scittle 0.8.32

Both workers run the identical hot loop (nested trig + sqrt over ~1800 particles). The only difference is the Scittle version. ms compute is the isolated cost of that loop; the JIT is close to 10× faster, which is why it holds ~60 fps while the interpreter drops into the 30s. ms draw (the canvas calls) is identical on both — a fair baseline.
